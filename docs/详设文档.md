# 希沃智教π · 详细设计文档

> AI智能作业批改系统 · 技术方案设计

---

## 一、系统概述

「希沃智教π」面向K12场景，以"拍照即批改、错题即诊断、评语即辅导"为核心体验，覆盖数学、语文、英语三大学科，实现从批改→诊断→辅导的完整闭环。

### 设计目标

| 目标 | 指标 |
|------|------|
| 批改准确率 | 客观题≥95%，主观题≥85% |
| 响应时间 | 单题批改<3秒，批量5题<10秒 |
| 个性化覆盖率 | 100%学生获得个性化评语 |
| 知识点诊断粒度 | 学科知识点级（200+节点） |

## 三、核心模块设计

### 模块1：多模态OCR识别引擎

**功能：** 将拍照上传的作业图片转为结构化文本

**处理流程：**
```
原始图片 → 预处理 → 区域检测 → 文字识别 → 结构化输出
   │         │         │          │          │
   │     倾斜校正    试题区域    手写体OCR   JSON格式
   │     噪声去除    答案区域    公式识别    题目+答案
   │     亮度增强    边界分割    符号识别    对应关系
```

**技术选型：**

| 组件 | 方案 | 理由 |
|------|------|------|
| 图片预处理 | OpenCV | 倾斜校正、去噪、增强 |
| 文字检测 | PaddleOCR DB | 中文场景最优 |
| 文字识别 | Qwen-VL / PaddleOCR | 多模态支持手写+公式 |
| 公式识别 | LaTeX-OCR | 数学公式转LaTeX |
| 区域分割 | YOLOv8 | 试题/答案区域检测 |

**创新点：**
- 混合识别策略：印刷体用PaddleOCR，手写体用Qwen-VL（多模态理解手写意图）
- 公式-文字混合排版解析：自动识别LaTeX公式与文字的边界

### 模块2：多学科批改引擎

**功能：** 对识别结果进行智能批改，输出分数、错误分析、知识点

#### 数学批改

```
解题过程解析 → 步骤拆分 → 逐步验证 → 错误定位 → 错误归因
```

**关键技术：**
- **分步解题验证**：LLM将学生解题过程拆分为独立步骤，逐步验证每步的正确性
- **过程分评估**（核心创新）：
  - 思路正确但计算失误：7-8分
  - 步骤部分正确、部分错误：4-6分
  - 思路完全错误但有尝试：1-3分
  - 自动识别"已掌握的步骤"并标注
- **错误类型分类**：
  - 概念错误（公式用错、定理用错）
  - 计算错误（运算失误、符号错误）
  - 方法错误（解题思路错误）
  - 表述错误（过程不完整、跳步）
- **正确解题生成**：自动生成标准解题过程供学生对照

#### 语文批改

```
文本理解 → 维度拆分 → 逐维度评分 → 段落点评 → 综合评价
```

**作文评分维度：**
| 维度 | 权重 | 评分标准 |
|------|------|----------|
| 立意 | 25% | 主题是否明确、是否有深度 |
| 结构 | 20% | 段落逻辑、过渡衔接 |
| 语言 | 25% | 用词准确性、句式丰富度 |
| 内容 | 20% | 素材恰当性、论证充分性 |
| 创新 | 10% | 观点独到性、表达新颖度 |

#### 英语批改

```
语法检查 → 拼写检查 → 时态检查 → 表达地道性 → 改进建议
```

**错误标注方式：**
```json
{
  "original": "I goed to school yesterday.",
  "errors": [
    {"word": "goed", "type": "不规则动词", "correction": "went", "rule": "go-went-gone"}
  ],
  "corrected": "I went to school yesterday."
}
```

### 模块3：知识图谱诊断引擎

**功能：** 将错题映射到知识点，诊断薄弱环节，追踪学习轨迹

**知识图谱结构：**

```
学科 → 章节 → 知识点 → 子知识点 → 考点
  │
  └─ 前置依赖关系
  └─ 常见错误关联
  └─ 难度等级标注
```

**数学知识图谱示例（部分）：**

```
代数
├── 整式
│   ├── 单项式
│   ├── 多项式
│   ├── 整式乘法 ← 常见错误：漏项、符号错误
│   └── 因式分解 ← 常见错误：提公因式不彻底、公式记错
├── 方程
│   ├── 一元一次方程 ← 前置依赖：整式运算
│   ├── 一元二次方程 ← 前置依赖：因式分解、整式乘法
│   │   ├── 求根公式法
│   │   └── 配方法
│   └── 不等式
└── 函数
    ├── 一次函数 ← 前置依赖：方程
    └── 二次函数 ← 前置依赖：一元二次方程、因式分解
```

**诊断算法：**

```python
def diagnose(results):
    """
    输入：多题批改结果
    输出：薄弱知识点 + 根因分析 + 学习建议
    """
    # 1. 收集所有错题的知识点
    error_points = []
    for r in results:
        if not r["is_correct"]:
            error_points.extend(r["knowledge_points"])

    # 2. 统计知识点错误频次
    point_freq = Counter(error_points)

    # 3. 追溯前置依赖
    root_causes = []
    for point, freq in point_freq.most_common():
        deps = get_prerequisites(point)  # 从图谱获取前置依赖
        for dep in deps:
            if dep in point_freq:
                root_causes.append({
                    "surface": point,
                    "root": dep,
                    "evidence": f"{point}错误可能源于{dep}基础不牢"
                })

    # 4. 生成学习路径
    learning_path = generate_learning_path(root_causes, point_freq)

    return {
        "weak_points": point_freq.most_common(5),
        "root_causes": root_causes,
        "learning_path": learning_path
    }
```

### 模块4：个性化评语引擎

**功能：** 基于学情档案生成"像班主任写的"个性化评语

**评语生成策略：**

```
学情数据 → 亮点识别 → 问题聚焦 → 建议生成 → 语气调整 → 最终评语
```

**个性化维度：**

| 维度 | 数据来源 | 作用 |
|------|----------|------|
| 历史成绩趋势 | 学情数据库 | 判断进步/退步/稳定 |
| 错误类型分布 | 批改结果 | 聚焦最突出的问题 |
| 学习风格 | 答题模式分析 | 调整建议方式 |
| 知识薄弱点 | 知识图谱 | 给出针对性建议 |

**评语示例对比：**

| 场景 | 通用评语（竞品） | 个性化评语（希沃智教π） |
|------|-----------------|----------------------|
| 数学全对 | "做得好，继续努力！" | "这次方程全部正确，说明因式分解已经掌握了。建议挑战一下配方法的题目，为二次函数做准备。" |
| 英语时态错 | "注意时态。" | "你已经能区分一般现在时和现在进行时了，但过去时的不规则动词还需要加强。建议整理一份go-went/see-saw的卡片，每天看一遍。" |
| 语文作文 | "内容不错，注意结构。" | "你的观点很独特，'每个人都是自己人生的作者'这个比喻用得特别好。建议在第二段加一个过渡句，让论述更连贯。" |

## 七、创新点总结

| 创新点 | 区别于常规方案 |
|--------|---------------|
| **诊断式批改** | 不只判对错，定位"哪一步出错+为什么错" |
| **过程分评估** | 识别"思路正确但计算失误"，按步骤正确比例给分，而非简单判对错 |
| **知识图谱链路追踪** | 知识点级诊断，追溯前置依赖根因 |
| **个性化评语引擎** | 基于学情档案生成"像班主任写的"评语 |
| **多学科差异化策略** | 数学分步验证、语文维度拆分、英语语法标注 |
| **教师可调的AI协审** | AI先批→教师复核→从修正中持续学习 |
| **学情累积画像** | 单次快照→连续趋势→预警推送 |
